Requirements

Get_Check_Total

Get_Check_Total.1
The system shall return the check total associated with the specified table index.

Get_Next_Party_To_Be_Seated

Get_Next_Party_To_Be_Seated.1
The system shall reset the waiting list index to 0 if it is greater than 9 before accessing the waiting list.
Get_Next_Party_To_Be_Seated.2
The system shall not reset the waiting list index if it is 9 or less before accessing the waiting list.
Get_Next_Party_To_Be_Seated.3
The system shall return the party at the current waiting list index and increment the index after returning the party.

Add_Included_Dessert

Add_Included_Dessert.1
The system shall set the dessert to PIE when the order includes STEAK as the entree, CAESAR as the salad, and MIXED_DRINK as the beverage.
Add_Included_Dessert.2
The system shall set the dessert to CAKE when the order includes LOBSTER as the entree, GREEN as the salad, and WINE as the beverage, and the first condition is not met.
Add_Included_Dessert.3
The system shall leave the dessert unchanged when the order does not meet either of the specified entree, salad, and beverage combinations.

Clear_Table

Clear_Table.1
The system shall mark the table as unoccupied and set the number in party to one when clearing a table.
Clear_Table.2
The system shall set all order fields for each seat at the table to their respective 'no' values when clearing a table.
Clear_Table.3
The system shall clear the order fields for exactly four seats when clearing a table.
Clear_Table.4
The system shall set the check total to zero, update the table record, and return a success indicator when clearing a table.

Add_Party_To_Waiting_List

Add_Party_To_Waiting_List.1
The system shall reset the waiting list index to zero when the waiting list is full before adding a new party.
Add_Party_To_Waiting_List.2
The system shall add a new party to the next available position in the waiting list when the list is not full.
Add_Party_To_Waiting_List.3
The system shall copy each character of a non-empty, non-NULL party name into the waiting list entry.
Add_Party_To_Waiting_List.4
The system shall not copy any characters if the provided party name is NULL or empty.
Add_Party_To_Waiting_List.5
The system shall ensure that each party name added to the waiting list is null-terminated.

Place_Order

Place_Order.1
The system shall retrieve the table record, mark the table as occupied, increment the party size, store the order for the specified seat, and add any included dessert as appropriate when an order is placed.
Place_Order.2
The system shall not add any amount to the check total when the entree is not ordered.
Place_Order.3
The system shall add 14.0 to the check total when the entree ordered is steak.
Place_Order.4
The system shall add 10.0 to the check total when the entree ordered is chicken.
Place_Order.5
The system shall add 18.0 to the check total when the entree ordered is lobster.
Place_Order.6
The system shall add 12.0 to the check total when the entree ordered is pasta.
Place_Order.7
The system shall not add any amount to the check total when the entree value does not match any defined entree.
Place_Order.8
The system shall update the table record with the new data and return a success indicator after processing the order.
